# Definitions

- **Expected State**: behavior and structure derived from SDD and architecture docs.
- **Current State**: behavior and structure proven from repository evidence.
- **Gap**: delta between expected and current state.
- **Gap Ledger**: master control document for gap status, priority, dependencies, and sequencing.
- **Implemented-Unverified**: implementation exists but required proof is incomplete.
- **Verified**: required proof passes and ledger can be updated truthfully.
- **Drifted**: implementation contradicts architecture or agreed system model.
- **Runnable Gap**: bounded, dependency-resolved gap with explicit proof requirements.
