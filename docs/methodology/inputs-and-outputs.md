# Inputs And Outputs

## Required Inputs

### Mandatory

- Software Design Document (SDD)
- Architecture and project document
- Current codebase
- Gap document with this schema:

| Area | Gap Item | Status | Reason | Remarks (Code-Grounded Assessment) | Priority | Depends On | Dependency | Execution Order |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |

### Optional But Strongly Recommended

- ADRs
- API contracts
- DB schema docs
- UX flows
- acceptance criteria
- test strategy
- coding standards

## Standard Outputs

GBD produces:

1. Derived Expected State
2. Code-Grounded Current State
3. Gap Ledger
4. Execution Plan per Gap
5. Tests per Gap
6. Implementation Evidence
7. Updated Gap Ledger
8. Release Readiness Summary
